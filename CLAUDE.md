# stock-dashboard — 專案記憶

## 使用者交易環境

- **券商＝國泰證券 app**。電子下單手續費很便宜（低折扣、零股單筆最低約 1 元）。
- **所以不要用「最低手續費 20 元會吃掉零股」當理由勸退小額/零股**；使用者用零股分批、買貴的股票（如南亞科）都可行，手續費占比極低。
- 台股有**盤中零股交易**，可買任意股數——策略試算不要假設「一定要買整張」。

## 時區（重要）

- **使用者時區 = 台灣 UTC+8（Asia/Taipei，無日光節約）。**
- **機器人與網頁顯示的所有時間、日期一律用 UTC+8。**
- 伺服器（GitHub Actions、Streamlit Cloud）跑在 **UTC**，所以**禁止**直接用
  `datetime.today()` / `datetime.now()` 產生要顯示或當清單日期的時間。
- 一律透過 `core/tz.py` 取現在／今天：
  - `now_tw()` → 現在的台灣時間（帶時區）
  - `today_tw()` → 今天的台灣日期
- 例外：`core/data.py` 內部走訪 TWSE 查詢日期也用 `now_tw()`，確保「今天是哪個交易日」以台灣日算。
- 看 GitHub Actions 的 run 時間戳記時記得那是 **UTC**，換算台灣要 **+8**。

## 排程（皆為台灣時間 UTC+8）

- 07:40 開盤前預測（主班）、08:10 備援班
- 15:20 收盤復盤、15:35 收盤後選股、18:00 復盤補跑
- 機器人內建備援排程 `jobs/bot.py` `_SCHED_SLOTS` 用 `_tw_now()`（= `now_tw()`）判時。

## 開發慣例

- 一人專案：直接開發、推 `main`，不開 PR。
- 改動後跑 `python -m pytest -q`。
- **修策略邏輯時：新規則一律「明天起才生效」，不回頭改已產出的結果**（`screen:latest`、已寫入的預測）。
  已存的快照/預測留著當「修正前」紀錄，下一次排程跑才套新邏輯；不重跑選股、不改 DB。

## 策略邏輯的坑（教訓）

- **「回檔承接」的進場判斷不能只看『靜態位置』，一定要看『當日那根 K 棒的動態（漲跌幅）』。**
  - 事件：藥華藥(6446) 當日**漲停 +10%**，卻被選股標成「進場（回檔到支撐、量縮）」。
  - 原因：四關（到支撐、止穩、量縮、外資停手）全是**靜態**條件，漲停會用**錯誤理由**踩中全部——
    ①強勢噴出時**短均線 MA5 被拉上來緊貼價格**→`|收盤−MA5|` 很小→誤判「到支撐」；
    ②`收盤≥昨收` 對 +10% 形同虛設→誤判「止穩」；③**漲停鎖死惜售、量自然縮**→誤判「量縮」。
  - 本質：**必要條件 ≠ 充分條件**；判「回檔」時沒排除「噴出」這個相反情境（方向/幅度盲）。
  - 修法：`core/rules.py` 加 `SURGE_PCT=4.5`，`entry_setup` 讀「當日漲幅」，≥門檻＝噴出非回檔→夾成觀望。
  - 通則：**日後任何「到均線/到支撐」的判斷，都要問「是價格跌過去、還是均線追上來？」**——用當日漲跌幅區分，別讓飆股每天都被判成「到支撐」。
